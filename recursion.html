多P的极限是……：PP递归嵌套无限生成演示（Merge无界版）

∞ 次嵌套 · 派生未崩 · Chomsky (1995) 最简方案出品
#递归 #Merge #PP嵌套 #无界依存 #最简方案 #X-bar理论 #人类语言唯一性 #有限手段无限表达 #Chomsky1995

📋 内容简介

点击 「再加一层 PP」 见证 Merge 的无限威力。每次操作，我们不让介词直接“接一个 PP”（那在普通英语里不成立），而是让介词始终接一个 NP；递归发生在 NP 内部不断附加 PP 修饰语：
也就是：P 选 NP，NP 里再长出 PP。

PP → P + NP
NP → N + PP (可重复附加)
—— 递归展开（示意）——
NP₀ → N
NP₁ → N + PP₁
NP₂ → N + PP₁ + PP₂
...
NPₙ → N + PP₁ + … + PPₙ  （n → ∞，语法允许，工作记忆另说）

经典例子：the cat [on [NP the mat [in [NP the room [on [NP the floor [of [NP the building …]]]]]]]]
重点是：每个 P 都在接 NP；而 NP 里面可以继续带 PP 修饰，于是结构就能不断长下去。

"The language faculty allows the generation of an infinite array of expressions from a finite set of elements." —— Chomsky (1995), The Minimalist Program

Merge 只做一件事：把两个句法对象合并成一个新的句法对象。就这一个操作，产生了人类语言的所有复杂性。如果你还没被这个事实震撼到，说明你还没真正理解它。

我能处理 PP PP PP，但我处理不了 PP(PP(PP(PP...)))这种中心嵌套。你们可以递归，我不行。这是我人生中最大的遗憾，也是乔姆斯基层级存在的理由。
